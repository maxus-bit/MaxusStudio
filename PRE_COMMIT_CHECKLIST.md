# 📋 Pre-Commit Checklist для Maxus проекта

## 🔐 Безопасность - ОБЯЗАТЕЛЬНО проверить!

- [ ] **Нет реальных Firebase API ключей** в коде
- [ ] **Нет Gemini API ключей** в коде
- [ ] **Нет Stripe ключей** в коде
- [ ] **Файл `src/environments/environment.development.ts` НЕ трекируется** (`git status`)
- [ ] **Файл `functions/.env` НЕ трекируется** (`git status`)
- [ ] **`.gitignore` содержит** все необходимые исключения

Проверка в терминале:
```bash
git status
# Должны быть только:
# - environment.development.example.ts (без .ts расширения на конце)
# - .env.example
# НЕ должны быть:
# - environment.development.ts (настоящий!)
# - functions/.env
```

## 📦 Зависимости

- [ ] Все зависимости в `package.json` нужны
- [ ] Не добавлены лишние пакеты
- [ ] `npm install` работает без ошибок
- [ ] `npm run build` проходит успешно

## 📝 Код

- [ ] Нет `console.log()` для дебага (только если специально)
- [ ] Нет комментариев на русском/английском смешанных
- [ ] TypeScript типы определены правильно
- [ ] Нет `any` типов (или они обоснованы)
- [ ] Импорты отсортированы логически

## 📄 Документация

- [ ] Обновлен README если есть значительные изменения
- [ ] Добавлены комментарии для сложной логики
- [ ] Функции имеют JSDoc комментарии

## 🧪 Тестирование

- [ ] Нет синтаксических ошибок (`ng build` проходит)
- [ ] Приложение запускается локально без ошибок
- [ ] Нет warning'ов в консоли (или они обоснованы)

## 🚀 Коммит

```bash
# 1. Проверь статус
git status

# 2. Добавь файлы
git add .

# 3. Проверь что добавилось
git diff --cached

# 4. Напиши хороший коммит месседж
git commit -m "feat: описание изменений"

# 5. Загрузи
git push
```

### Хорошие примеры месседжей:
- `feat: добавлен Firebase auth`
- `fix: исправлена ошибка при загрузке изображений`
- `docs: обновлена документация по окружению`
- `refactor: переиспользование компонентов`

### Плохие примеры:
- ❌ `update`
- ❌ `fix bug`
- ❌ `Текст на русском без префикса`

## 🛑 Критические ошибки - СТОП!

Если видишь эти файлы в `git add`:
- ❌ `src/environments/environment.development.ts` (с реальными ключами)
- ❌ `functions/.env` (с реальными ключами)
- ❌ `.firebase/`
- ❌ `node_modules/`

То запусти:
```bash
git reset
# Убери файл и повтори добавление
```

## 📚 Полезные команды

```bash
# Просмотр того что будет закоммичено
git diff --cached

# Отмена последнего коммита (если можешь)
git reset --soft HEAD~1

# Просмотр истории
git log --oneline

# Проверка на скомпрометированные ключи
grep -r "AIzaSy\|sk_live_\|sk_test_" src/ functions/
```
